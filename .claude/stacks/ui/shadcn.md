# UI: shadcn/ui
> Used when idea.yaml has `stack.ui: shadcn`

## Packages (devDependencies)
```bash
npm install -D tailwindcss @tailwindcss/postcss
```

## Setup Commands
Run in order:
1. `npx shadcn@latest init --yes --defaults` — sets up Tailwind, PostCSS, and globals.css (the `--yes --defaults` flags prevent interactive prompts)
2. `npx shadcn@latest add -y button input card label separator` — install base components (add more later as needed)

## Post-Setup Verification
After running setup commands, verify:
- `postcss.config.mjs` uses `@tailwindcss/postcss` — fix if shadcn generated a different configuration
- `src/app/globals.css` contains `@import "tailwindcss"` — fix if missing
- `package.json` still has `dev`, `build`, `start`, and `lint` scripts — re-add if shadcn overwrote them

## Component Directory
- `src/components/ui/` — auto-generated by shadcn
- Do not manually edit files in this directory

## Adding Components Later
```bash
npx shadcn@latest add -y <component>
```

## Theme Setup

After `shadcn init` completes and post-setup checks pass, apply these customizations:

### 1. Font (in layout.tsx — created in Step 3)
Import the chosen font(s) from `next/font/google` (ships with Next.js — no extra package). When the frontend-design plugin is active, use the font it selects. When no plugin guidance is available, default to Inter.
- Example: `const displayFont = Inter({ subsets: ["latin"], variable: "--font-sans" })`
- Apply `className={displayFont.variable}` to the `<html>` tag
- If using a display + body font pair, apply both variables: `className={\`${displayFont.variable} ${bodyFont.variable}\`}`

### 2. CSS overrides (in globals.css — exists after shadcn init)
After the shadcn-generated `:root` block, add:
- `--primary: 221 83% 53%` (or domain-appropriate hue per design.md Section B)
- `--primary-foreground: 0 0% 100%`
- `--radius: 0.5rem`
- Match `.dark` block with lighter primary variant

### 3. Post-theme verification
- globals.css contains `--primary` override
- layout.tsx imports the display font (plugin-selected or Inter default) and applies variable to `<html>`

## Patterns
- Always use library components (`<Button>`, `<Input>`, `<Card>`, etc.) — don't mix with raw HTML elements (`<button>`, `<input>`)
- Follow `.claude/patterns/design.md` for all visual decisions (spacing, typography, component sizing)
- Import components from `@/components/ui/<component>` — these are auto-generated by shadcn
- When a new component is needed, add it with `npx shadcn@latest add -y <component>` rather than building from scratch

## Import Example
```tsx
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";
```
